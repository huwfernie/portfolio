Node projects

I also have here two projets that I have worked on recently. Both are primarily Node.js and are more like tools than MVP's. Even though they have very specific use cases; buliding them both was a challenge, and gave me some nice problems to solve.

Yabee

Yabee; is Ebay spelled bakwards and with an extra 'e' added to help with pronouciation. It's the last project I did during my training at General Assembly and went from conception to this MVP in just over a week.

The idea was to create a version of Ebay and Gumtree where it was also possible to list yourself, as a person willing to buy something. This would give sellers the option of a much quicker sale than waiting for an auction to end or for someone to respond to an add.

The Stack

The styling was taken from Gumtree.com, and at the time it was made many layout features would match to the pixel. The front end is Angular, while the back end is Ruby on rails talking to an SQL database.

Messenger

I wanted the sellers to be able to talk with potential buyers, so I also made a simple message service within the app which works well.

Given the short sprint time this took a disproportionate amount of time to impliment.

Travel Blogging

The idea for this site was to try to take some of the pain out of setting up and maintaining a personal blog, as well as being a single point to visit when looking for a bit of inspiration. So a cross between sites like www.instructables.com and www.medium.com

The Stack

This site is built on a Node backend, using Express for the framework and EJS as the view engine. Blogs and users are saved in a NoSQL database.

Features

To help with presentation the site incorporates a google map with pins for each of the blogs. It also has facebook/twitter style profile pages. Blogs are written in markdown which lets the authors add some basic formatting and images can be added to blogs - these are stored on AWS.

You can see the final result here Travel Blog

Or you can look through the code here: GitHub.

City Running

Run-IMG was a group project, 4 peoples work from a week long sprint. The idea was to map out running routes in London.

The start and end point for each route would be a recognisable monument or landmark. To register a run you need to submit a picture at the start and finish of the track which we use to verify and time the run.

That picture is then sent to the Google Vision API, which can confirm the co-ordinates of where the photo was taken. Now a timer can start and we put in Google Maps to give you an end point to race to.

This was a fun app to build, and we felt that with a bit of tweaking and by cleverly integrating social media it could have gone somewhere.

The Stack

This is a MEAN stack app: Angular and jQuery on the frontend, with Mongo, Express and Node on the backend. As it was designed to be used on the go, all the styling was for mobile first, and as we were short on time a desktop version was never built.

The app is online: Run IMG
*However registration for Google Vision has expired, so their is limited functionality.

Or you can look through the code here: GitHub.

Blog to Kindle

Say you sit at a computer all day, also lets imagine you like reading… but after work you really don't want to read stuff on another LCD screen. Its' great that they invented e-books like Kindle for people just for you.

But, what if the thing you really want to read is online, like a blog, something like Mr Money Moustache wouldn't it be great if you could turn a blog into an e-book?

The Stack

This site is really a follow on from my work related scrape from live project. It's all Javascript working in Node, there is a simple HTML form and then a button to start the process.

Localhost

To begin with, node will start a localhost, then the form asks for some details about the blog, some are for the kindle file, the important ones are the target web address.

Process

Node can then load the first blog. Using some pre-programmed (but easily changeable) CSS selectors it's possible to strip out the body of the blog. It's also possible with CSS to find the little button inviting you to read the next blog… so blog 1 leads to 2, to 3… either until a limit is reached, or until there is no "next" button on the page.

Command Line

When all the blogs have been scraped, the titles need to be stripped out into a table of contents, and a bulid manifest has to be made. Then these files are sent into a command line tool made by Amazon, which - if everything has worked properly - will output a kindle file with the whole blog, saved as one big book.

You can download a sample book: here.

There are still some bugs in the code so the final step doesn't work for now!

If you want to help out with the code have a look here: GitHub.

Scrape from live:

This started out as a tool to make work a little bit easier; let's set the scene. We're working for an agency that has to update many sites, the new design is mostly done but you need to put in a few bits of information from the current live site.

Of course it is possible to do this by looking through all the files in the repo, but wouldn't it be nice to have one refence file, definitely up to date, and with just the sections you're interested in so you don't end up searching through hundreds of files.

The Stack

This is really a few JavaScript functions running in Node. It needed a tiny bit of user input, so that was a small HTML form, with a tiny bit of CSS - it was never designed to be pretty!

Features

Once you run the Node app, you can go to the localy hosted homepage where you find a form.

Enter details about the website you want to scrape, and what country codes you need.

Figure out the CSS selector that will get you what you need.

Click "go", and look for any progress/errors in the terminal.

Once it's run, click "finish" and open the timestamped html file to see the results.

* this was built to work with some specific clients, for their privacy I've changed this code to work with Wikipedia instead.

You can see an example scrape from Wikipedia here: Scrape from Wikipedia
